## 一、论文概述

DeepSeek-R1 是深度求索（DeepSeek）发布的第一代推理模型，核心贡献在于**证明了大语言模型的推理能力可以通过纯粹的强化学习（RL）来激励，无需依赖人工标注的推理轨迹**。该论文在 Nature 正刊发表，标志着其在学术界和工业界的重大影响力。

论文提出了两个核心模型：
- **DeepSeek-R1-Zero**：基于纯强化学习训练（无SFT），验证了RL激励推理能力的可行性
- **DeepSeek-R1**：结合冷启动数据 + 多阶段训练流程，性能对标 OpenAI o1-1217

### 3.2 核心算法：GRPO（Group Relative Policy Optimization）

GRPO 是本论文的核心 RL 算法，相比传统 PPO 有显著创新：

**传统 PPO vs GRPO 对比：**

| 特性 | PPO | GRPO |
|:---|:---|:---|
| Critic 模型 | 需要（与策略模型同等规模） | **不需要** |
| 优势函数估计 | GAE（基于Critic） | **组内相对奖励** |
| 训练成本 | 高（需维护Critic） | **低（大幅节省内存和计算）** |
| 基线估计 | 学习的价值函数 | **同组采样的平均奖励** |

**GRPO 核心公式：**

对于每个问题 q，从旧策略 π_θ_old 中采样一组输出 {o₁, o₂, ..., o_G}，然后优化目标函数：

```
J_GRPO(θ) = E[ 1/G Σᵢ ( min(rᵢ(θ)·Aᵢ, clip(rᵢ(θ), 1-ε, 1+ε)·Aᵢ) - β·D_KL(πθ||πref) ) ]

其中:
  rᵢ(θ) = πθ(oᵢ|q) / πθ_old(oᵢ|q)   (策略比率)
  Aᵢ = (rᵢ - mean(R)) / std(R)         (组内相对优势)
  R = {r₁, r₂, ..., r_G}               (组内奖励集合)
```

**关键设计思想**：
- 不使用神经奖励模型，避免 Reward Hacking
- 采用基于规则的奖励：
  - **准确性奖励**：数学题验证 `\boxed{}` 答案 / 代码执行测试用例
  - **格式奖励**：要求推理过程包含在 `<think>...</think>` 标签中

### 3.4 "Aha Moment"（顿悟时刻）

论文描述了一个极其引人注目的现象——模型在RL训练过程中出现了类人的"顿悟时刻"：

> 在训练的中间阶段，R1-Zero 学会了重新审视自己的初始解题思路。它会在推理过程中突然说出类似 **"Wait, wait. Wait. That's an aha moment I can flag here."**（等等等等。这是一个我可以标记的顿悟时刻）这样的表述，然后重新组织思路。

**示例推理片段**：
```
<think>
...（前面的推理步骤）...
Wait, wait. Wait. That's an aha moment I can flag here.
Let's reevaluate this step-by-step to identify if the correct sum can be...
...（重新推理，找到正确答案）...
</think>
```

> **研究者感言（原文）**："This moment is not only an 'aha moment' for the model but also for the researchers observing its behavior. It underscores the power and beauty of reinforcement learning: rather than explicitly teaching the model on how to solve a problem, we simply provide it with the right incentives, and it autonomously develops advanced problem-solving strategies."

**涌现的复杂推理行为包括**：
- 🔄 **自我反思（Self-reflection）**：重新审视之前的推理步骤
- ✅ **自我验证（Self-verification）**：主动检查答案的正确性
- 🔀 **动态策略适应**：尝试不同的解题方法
- ⏱ **自适应思考时间**：根据问题难度调整推理长度

## 四、DeepSeek-R1 四阶段训练详解

### 阶段1：冷启动 SFT

**目的**：解决R1-Zero的可读性问题，为RL提供稳定起点

- 收集**数千条**高质量长思维链（CoT）数据
- 数据来源：
  - R1-Zero 输出的可读版本
  - 人工标注和修正
  - Few-shot 提示生成
- 定义清晰输出格式：`|special_token|<推理过程>|special_token|<答案总结>`

### 阶段2：推理导向RL训练

**目的**：专注提升数学、代码、科学、逻辑等推理能力

- 使用与 R1-Zero 相同的 GRPO 算法
- **创新点**：引入**语言一致性奖励**
  - 计算推理链中目标语言词汇的比例
  - 惩罚语言混用行为
  - 最终奖励 = 准确性奖励 + 语言一致性奖励

### 阶段3：拒绝采样 + SFT

**目的**：扩展模型的通用能力

| 数据类别 | 数量 | 来源 |
|:---|:---:|:---|
| 推理数据 | ~60万条 | RL checkpoint 拒绝采样（仅保留正确答案） |
| 非推理数据 | ~20万条 | DeepSeek-V3 SFT 数据（写作、角色扮演、翻译等） |
| **合计** | **~80万条** | 在 V3-Base 上微调 2 epochs |

**数据处理策略**：
- 过滤混合语言、长段落冗余、代码块混乱的输出
- 部分任务使用 DeepSeek-V3 作为生成式奖励模型判断正确性
- 简单查询（如"你好"）不添加 CoT

### 阶段4：全场景RL

**目的**：对齐人类偏好，确保有用性和无害性

```
                  ┌── 推理任务 → 基于规则的奖励 (数学/代码/逻辑)
全场景RL ─────────┤
                  └── 通用任务 → 奖励模型 (有用性 + 无害性)
                        │
                        ├── 有用性评估：仅评估最终总结
                        └── 无害性评估：评估完整响应（推理过程 + 总结）
```

## 六、失败尝试与经验教训

论文坦诚分享了两个不成功的探索方向：

### 6.1 过程奖励模型（PRM）

**尝试原因**：PRM 可以逐步引导模型走向更好的推理路径

**失败原因**：
1. 难以为通用推理定义细粒度的"步骤"
2. 判断中间步骤是否正确本身就很困难
3. 引入基于模型的 PRM 不可避免地导致 Reward Hacking
4. 相比带来的收益，额外计算开销过大

### 6.2 蒙特卡洛树搜索（MCTS）

**尝试原因**：受 AlphaGo/AlphaZero 启发，系统性探索解空间

**失败原因**：
1. 与棋类游戏不同，token生成的搜索空间呈指数级增长
2. 设置节点扩展上限导致陷入局部最优
3. 训练细粒度价值模型本身就极度困难
4. AlphaGo 的核心——价值模型迭代提升——在token生成场景中难以复现

## 七、当前局限性与未来方向

| 局限性 | 描述 | 未来计划 |
|:---|:---|:---|
| **通用能力不足** | 函数调用、多轮对话、复杂角色扮演、JSON输出等不如V3 | 探索长CoT增强这些任务 |
| **语言混杂** | 仅针对中英文优化，其他语言可能用英文推理回答 | 扩展多语言支持 |
| **Prompt敏感** | Few-shot会降低性能，推荐Zero-shot | 提升鲁棒性 |
| **软件工程** | 因评估耗时，RL未大规模应用于工程任务 | 引入拒绝采样/异步评估 |
